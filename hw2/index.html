Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

The De Casteljau algorithm is a recursive method for evaluating points on a bezier curve at a given parameter t \( ∈ [0,1] \), and n+1 control points. The algorithm repeatedly performs the following equation \(p_i' = (1-t)*p_i+t*p_{i+1}\), reducing the number of points until we get to a single point on the bezier curve at t. In our implementation, we iterate through the input vector of points and compute each point using the formula described earlier, using the public parameter t, pushing the result into a new Vector2D vector. This returns an n-1 vector, allowing us to step through each step of the algorithm until we get to a single point, which the the point on the curve.

Part 2: Bezier surfaces with separable 1D de Casteljau

De Casteljau's algorithm extends to Bezier surfaces by applying the 1D algorithm twice in a separable fashion. Given an nxn grid, and two parameters u and v, we first apply the algorithm along each row of control points using u as our parameter, reducing each row to a single 3D point. This results in n points, one per row. Then we apply the algorithm again using parameter v, resulting in a single point on the surface at (u,v). In our implementation, the application of the De Casteljau algorithm at each individual row is the same, but we have two outer functions, evaluate 1D and evaluate. evaluate1D accepts a row and a parameter, and repeatedly calls on the evaluateStep, which is identical to the function in part 1, until a single point remains, and returns that point. evaluate calls evaluate1D repeatedly on every row with parameter u, and stores the points in a vector, it then calls evaluate1D one last time with parameter v, and returns a single point on the curve at (u,v).

Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

To compute area-weighted vertex normals, we iterate through all faces incident to the vertex using the half-edge data structure. Starting from the vertex's half-edge, we traverse to each neighboring face by repeatedly following twin()->next(). For each face, we retrieve its three vertex positions (p0, p1, p2) and compute the cross product of two edge vectors (p1-p0) and (p2-p0). The magnitude of this cross product is proportional to the triangle's area, so summing these cross products directly gives us the area-weighted normal sum. Finally, we return the unit vector of this sum as the normalized area-weighted vertex normal.

Part 4: Edge flip

To implement the edge flip operation, we first check if the edge is a boundary edge and return immediately if so. Otherwise, we begin by collecting all relevant mesh elements — the six half-edges, four vertices, five edges, and two faces involved in the two triangles sharing the edge. We then reassign all pointers for every element in the modified mesh, even those that did not change, to avoid any missed pointer bugs. The key change is that the shared edge is redirected to connect the two previously unconnected vertices of the pair of triangles, and the half-edges of both faces are updated accordingly to reflect the new triangle configurations. Vertex and face half-edge pointers are also updated to ensure consistency throughout the mesh. Part 5: Edge split

To implement the edge split operation, we first check if the edge is a boundary edge and return immediately if so. Otherwise, we collect all existing mesh elements — the six half-edges, four vertices, four edges, and two faces of the two triangles sharing the edge. We then allocate all new elements needed: one new vertex at the midpoint of the original edge, three new edges, two new faces, and six new half-edges to connect everything together. The original edge and one of its two faces are reused rather than deleted and recreated. As in Part 4, we exhaustively reassign all pointers for every element in the modified mesh, including those that did not change, to prevent subtle bugs from missed assignments. Part 6: Loop subdivision for mesh upsampling

Loop subdivision is implemented in three main steps. First, before any topology changes, we compute the updated positions for all existing vertices and edge midpoints using the original mesh geometry. For each old vertex, the new position is computed as a weighted average of its original position and the sum of its neighbors' positions, using the Loop subdivision weights based on vertex degree. For each original edge, the midpoint position is computed as a weighted combination of the four vertices of the two triangles sharing that edge. Second, we perform 4-1 subdivision by first snapshotting all original edges into a separate list, then splitting each of them in order. Snapshotting is critical — iterating directly over the mesh's edge list while splitting would cause newly created edges to be split as well, resulting in an infinite loop. Each newly inserted midpoint vertex receives the pre-computed edge midpoint position. After all splits, we iterate over all edges and flip any new edge that connects an old vertex to a new vertex, completing the 4-1 subdivision pattern. Finally, all vertex positions are updated from their stored new positions. Sharp corners and edges tend to smooth out and shrink inward with each iteration of loop subdivision, as the weighted averaging pulls vertices toward their neighbors. This effect can be partially mitigated by pre-splitting edges around sharp features, which increases the local vertex density and gives the subdivision algorithm less room to smooth aggressively. The cube becomes asymmetric under repeated subdivision because the original mesh has diagonals running in one direction per face, biasing the subdivision. Pre-processing by splitting each face diagonal into an X pattern makes the mesh rotationally symmetric, so the cube subdivides evenly across all faces. However, this introduces a tradeoff — the center vertex created by each X has a higher degree than the surrounding vertices, causing it to be pulled inward more aggressively by the weighted averaging, resulting in a slight divot at the center of each face.
